# 작업지시서 — 학급 투표 앱

## 화면 목록 (등장 순서)

### 1. 시작 화면
- 요소: "교사" 버튼, "학생" 버튼 (큰 버튼 2개, 밝고 화려한 톤 — 원색 계열, 눌렀을 때 애니메이션)
- 동작: 교사 클릭 → 2번 화면 / 학생 클릭 → 4번 화면

### 2. 교사 비밀번호 화면
- 요소: 비밀번호 입력창, 확인 버튼
- 동작: 맞으면 3번 화면으로. 틀리면 안내 문구 띄우고 다시 입력
- 비밀번호는 config.js 또는 코드 내 상수로 관리 (학생 개인정보 아님, 교사 1명용 고정값)

### 3. 교사 화면 — 질문 등록 & 실시간 집계
- 상단: 질문 등록 폼
  - 질문 텍스트 입력창
  - 선택지 입력 (기본 2개, "+선택지 추가" 버튼으로 자유롭게 늘림, 최소 2개 유지)
  - "투표 시작" 버튼 → 이 질문이 학생 화면에 열림
- 하단(또는 탭): 현재 진행 중인 질문의 실시간 집계 결과
  - 선택지별 득표 수 + 막대 비율
  - Supabase realtime 구독 또는 폴링으로 갱신
- 이미 진행 중인 질문이 있을 때 "투표 시작" 버튼은 비활성화하거나 "마감 후 새 질문" 흐름으로 안내

### 4. 학생 화면 — 투표
- 요소: 현재 열려 있는 질문 + 선택지 버튼들
- 진행 중인 질문이 없을 때: "진행 중인 투표가 없어요" 대기 화면
- 이미 이 기기로 이 질문에 투표했을 때: 선택지 대신 "투표 완료" 표시 + 5번 화면(결과)으로 자동 전환
- 선택지 클릭 → 투표 기록 저장 → 5번 화면으로 전환

### 5. 결과 화면 (교사/학생 공통 컴포넌트)
- 선택지별 득표 수와 비율을 막대그래프 형태로 표시
- 실시간 갱신 (새 투표가 들어오면 화면이 자동으로 업데이트)
- 학생 화면에서도 동일한 컴포넌트 재사용

## 데이터 (Supabase — vote_records 테이블)

| 컬럼 | 설명 |
|---|---|
| id | 자동 생성 PK |
| question_id | 질문 식별자 (질문 등록 시 생성) |
| question_text | 질문 내용 |
| option_text | 선택한 항목 텍스트 |
| device_id | 기기 식별용 임의 값 (localStorage에 생성해 저장, 개인정보 아님) |
| created_at | 투표 시각 |

- 개인정보(이름/번호/학번) 컬럼 없음 — 완전 익명
- 중복 투표 방지: 학생 화면 진입 시 localStorage에 `device_id` 없으면 생성. 투표 전 `question_id + device_id` 조합이 이미 있는지 확인 → 있으면 재투표 차단하고 결과 화면으로

## 데이터 (Supabase — current_question 테이블, 3단계에서 추가)

vote_records에는 선택지 목록 컬럼이 없어, 아직 아무도 투표하지 않은 새 질문(과 선택지 목록)을 학생 화면에 알릴 방법이 없다. 이를 위해 테이블을 하나 더 둔다.

| 컬럼 | 설명 |
|---|---|
| id | 자동 생성 PK |
| question_id | 질문 식별자 (vote_records.question_id와 동일 값으로 매칭) |
| question_text | 질문 내용 |
| options | 선택지 목록 (jsonb 배열) |
| active | 현재 진행 중 여부 (boolean) |
| created_at | 등록 시각 |

- 학생/교사 화면은 `active = true`인 행 중 가장 최근 것을 "현재 질문"으로 사용
- 교사가 "마감하고 새 질문 등록"을 누르면 해당 행의 `active`를 `false`로 갱신 (행 삭제 아님, 이력 보존)
- 새 질문을 시작하면 새 행을 insert (active: true)

## 엣지 케이스
- config.js가 없거나 키가 비어 있을 때: 화면 상단에 "Supabase 설정이 필요해요" 배너 표시 (마일스톤 3~4 구간에서는 정상 상태)
- 진행 중인 질문이 없을 때: 학생 화면은 대기 안내만 표시
- 같은 기기로 같은 질문에 두 번 투표 시도: 두 번째는 차단, 결과 화면으로 안내
- 인터넷 연결 끊김/Supabase 오류: 결과 화면에 "연결이 원활하지 않아요, 잠시 후 다시 시도" 안내

## 완성 기준 체크리스트
- [ ] 시작 화면에서 교사/학생 진입 분기가 동작한다
- [ ] 교사 비밀번호 확인이 동작한다
- [ ] 교사가 선택지 2개 이상인 질문을 자유롭게 등록할 수 있다 (2지선다~다지선다 모두)
- [ ] 학생이 투표하면 기기당 1회만 반영된다 (같은 기기 재투표 차단)
- [ ] 교사·학생 화면 모두 실시간으로 결과가 갱신된다
- [ ] 개인정보 컬럼이 어디에도 없다
- [ ] 모바일 화면(최대 너비 600px)에서 레이아웃이 깨지지 않는다
